Diligence Agent · FinanceBench · Kimi-K2.6 on Vultr · July 4–5, 2026

Five iterations of run → analyze → improve, measured to the item

An autonomous improvement loop ran the full 61-question eval five times, with Opus analyst and implementer agents between runs. Correct answers rose from 22 (naive-RAG baseline) and 37 (first agent run) to 51 of 61, while worst-case item latency fell from 634s to under 100s. Roughly half of the apparent gains came from fixing the measurement, not the agent — every scoring change was audited, and all runs were re-scored under the final scorer so this trend is apples-to-apples.

Measurement integrity

scoring versions v0 → v4 · every change audited, all runs re-scored

The deterministic scorers were never weakened: each version adds a pass path gated on a new human-reviewable gold annotation, derived from the gold answer text alone — never from agent output. The chart shows the same two frozen runs re-scored under each scorer version: the runs didn't change; the ruler did.

The audit trail
  • v1 parsed 9 numeric golds ("$8,400,000,000" → 8 400 USD millions) so the ±1% numeric scorer applies.
  • v2 added polarity/canonical branches over 23 annotations; the adversarial audit verified each derives from gold text; the one genuinely-wrong polarity item (jj_06) was annotated so it keeps failing.
  • v3 added 6 annotations after the coverage round, including verizon_05 — flagged, then human-signed-off (the gold's own formula is "1097 + 862").
  • v4 added one generic geography synonym and two entity-list annotations.
  • Guard in action: a proposed pepsico_03 annotation ("North America") was reverted — the gold lists multiple regions, so crediting it would have masked a genuinely incomplete answer.

The judges: from flat 5.0 to real signal

Tier-2 LLM judges · 1–5 rubrics behind a calibration gate
What it took
  • Iterations 1–2 scored a flat 5.0 with zero variance across all 61 items — the rubric couldn't discriminate, and Kimi's hidden reasoning channel was silently truncating responses to empty strings.
  • Fixes: anchored rubrics with contrastive exemplars, a period-mismatch hard-negative in the calibration gate, a 16k token ceiling, coverage reporting, and a gold-agreement criterion (the judge may see gold — it's eval-side).
  • A bake-off then moved judging to DeepSeek-V4-Flash: calibration gate pass, 9/9 score agreement with Kimi on sampled items, ~30× faster (the judge pass fell from ~60 to ~12 minutes).
  • Gold-agreement now tracks the deterministic scorer and is the only quality trend for judgment-bucket answers.

What's still wrong, and what iteration 6 would do

the 10 non-correct items of iteration 5

Three themes: churn (four items flip with sampling/retrieval variance even at temperature 0 — the fix is self-consistency or answer-form post-processing, not prompts), two real reasoning defects (tax-rate sign on a pre-tax loss; growth-basis selection), and two contested golds that need dataset review rather than agent work. Citation precision (58%) is the other open front: the all-or-nothing scorer punishes any extra citation, and trimming over-citation further conflicts with the multi-evidence habit that makes answers right.

Method. 61 curated FinanceBench questions across 11 companies (34 multi-input / 14 judgment / 13 lookup), superset of the original 28. Each iteration: full agent run (one process per company, 6-way parallel) → deterministic scoring (accuracy, citation precision/provenance, arithmetic integrity, trace shape, abstention) → LLM judges behind a corrupted-memo calibration gate → Opus analyst workflow → ranked improvement plan → Opus implementer workflow with adversarial verification → pilot → next run. Agent model: moonshotai/Kimi-K2.6 (Vultr). Instrumentation added in phase 0: per-call tokens/latency (llm_calls.jsonl), per-item stage waterfalls, failure taxonomy.

Artifacts. results/iterations/iter{1..5}/ (staged runs, per-item scores, metrics, plans, scoring-version snapshots) · results/iterations/report_data.json · scripts/run_iteration.py · scripts/analyze_iteration.py · data/gold_annotations.json (every annotation carries its gold-text evidence).
